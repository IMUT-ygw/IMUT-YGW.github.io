深入理解Java虚拟机

学习
Jvm
发布日期:   2020-02-02

简介

JVM是Java Virtual Machine（Java虚拟机）的缩写，JVM是一种用于计算设备的规范，它是一个虚构出来的计算机，是通过在实际的计算机上仿真模拟各种计算机功能来实现的。

JVM的位置

系统虚拟机模拟硬件部分，程序虚拟机例如jvm在操作系统之上。

JVM的整体结构

细化后

JVM架构模型

一种是基于栈的指令集架构，另一种是基于寄存器的指令集架构

区别

基于栈式指令集的特点

⚪设计和实现更简单，适用于资源受限的系统（打印机、自主独立的机器）

⚪使用零地址指令分配的方式（无地址）

⚪不需要硬件支持，可以执行好，更好的实现跨平台性

基于寄存器指令集的特点

⚪典型应用x86的二进制指令集，比如传统的PC以及Android的Davlik虚拟机。

⚪指令集架构依赖于硬件，可移植性差

⚪性能优秀执行更高效

⚪大部分情况下，基于寄存器架构的指令集往往都是以一地址指令、二三地址指令为主。

由于跨平台性的是设计，java的指令集都是基于栈式的指令集架构，因为不同平台的cpu架构不同，所以不能设计为基于寄存器架构的指令集，优点是跨平台性，指令集小，编译器容易实现。缺点是性能下降，实现同样的功能指令会更多。

JVM生命周期

虚拟机的启动

​ Java虚拟机的启动时通过引导类加载器创建一个初始类来完成的。每个类的父类都是Object类。类的加载器首先加载Object类，也就是根类加载器加载，运行main方法，进入main方法后再去执行其他类加载器。

虚拟机执行

​ 执行一个 所谓的java程序的时候，真真正正在执行的是一个叫做java虚拟机的进程。

虚拟机退出

​ 🌂 程序正常执行结束

​ 🌂 程序在执行时遇到异常或错误而终止

​ 🌂 可在代码中调用Runtime类的halt方法或System类的exit方法直接断开进程

类加载子系统

类加载子系统负责从文件系统或者网络中加载Class文件，class文件在文件开头有特定的文件标识。它只负责加载Class文件，至于他是否运行则由执行引擎决定。

类加载到内存后会生成一个大的Class实例，加载的类信息存放在一块名为方法区的内存空间。除了类的信息外，方法区还会存放运行时常量池信息，可能还包含字符串字面量和数字字面量。

简述

双亲委派机制

Java虚拟机对class文件采用的是按需加载的方式，也就是说当需要使用该类时才会将他的class文件加载到内存生成class对象。而加载某个类的class文件时，Java虚拟机采用的是双亲委派模式。

工作原理

如果一个类加载器收到类加载请求，他并不会自己先去加载这个类，而是把这个请求委托给父类的加载器去执行。

如果父类加载器还存在父类，则进一步向上委托，依次递归，请求最终将到达到顶层的启动的类加载器。

如果父类加载器可以完成加载任务，就成黄金功返回。如果父类加载器无法完成此任务，子类加载器才会尝试自己去加载。

加载流程：

  1. 当AppClassLoader加载一个class时，它首先不会自己去尝试加载这个类，而是把类加载请求委派给父类加载器ExtClassLoader去完成。
  2. 当ExtClassLoader加载一个class时，它首先也不会自己去尝试加载这个类，而是把类加载请求委派给BootStrapClassLoader去完成。
  3. 如果BootStrapClassLoader加载失败（例如在$JAVA_HOME/jre/lib里未查找到该class），会使用ExtClassLoader来尝试加载；
  4. 若ExtClassLoader也加载失败，则会使用AppClassLoader来加载，如果AppClassLoader也加载失败，则会报出异常ClassNotFoundException。

优势

避免类的重复加载（只要有类加载器去加载，其他类加载器就不会加载）

保护程序安全，防止核心API被篡改。

沙箱安全机制

沙箱安全机制是由基于双亲委派机制上 采取的一种JVM的自我保护机制,假设你要写一个java.lang.String 的类,由于双亲委派机制的原理,此请求会先交给Bootstrap试图进行加载,但是Bootstrap在加载类时首先通过包和类名查找rt.jar中有没有该类,有则优先加载rt.jar包中的类,因此就保证了java的运行机制不会被破坏.

运行时数据区

PC寄存器（程序计数器）

cpu中的寄存器存储指令相关的现场信息、cpu只有把数据装载到寄存器才能够运行。pc寄存器并非cpu中的物理寄存器，它是一种软件形式模拟硬件的一种抽象模拟。但要问起来，jvm中含有寄存器结构。

作用:

PC寄存器用来存储指向下一条指令的地址，执行引擎通过地址读取指令，并执行。

他是一块很小的内存空间，几乎可以忽略不计，也是运行速度最快的存储区。

在jvm规范中，每个线程都有自己独立的程序计数器，是线程私有的，生命周期与线程生命周期一致。

任何时间，一个线程都有一个方法在执行，程序计数器会存取当前线程正在执行的java方法的jvm指令地址，或者如果执行的是native方法，则是未指定的值（undefined）因为程序计数器是java层面的，方法调用本地c或c++会显示不出来。

寄存器中没有GC 和 OOM。

pc寄存器存储指令地址，执行引擎拿到偏移地址并得到操作指令，将一系列的操作交给局部变量表，操作数栈实现数据的存取等操作。并要把字节码指令转化为机器指令，利用cpu进行运算。

面试问题

问题一：使用pc寄存器存储字节码指令有什么用？或 为什么要用pc寄存器记录当前线程的执行地址？

答：因为cpu在工作时时不断切换线程的，每个线程都有一个程序计数器，当cpu切换回来时就可以通过程序计数器，知道该从什么地方继续执行。

问题二：pc寄存器为什么会被设定线程私有？

答：为了能够准确的记录各个线程正在执行的当前的字节码指令地址，最好的方法就是为每一个线程都要分配一个程序计数器。由于cpu时间片轮转的限制，众多线程在并发执行的过程中，任何一个确定的时刻，一个处理器或多核处理器的中的一个内核，只会执行某个线程的一条指令。这样必然会导致经常终端或恢复，线程与线程间相互独立，必须每个线程都有自己的程序计数器。

虚拟机栈

内存中的栈和堆

栈解决运行问题，即程序如何执行，或者说如何处理数据。

堆解决数据存储问题，数据怎么放，放在哪。

Java虚拟机栈，早期也叫Java栈。每个线程创建时都会创建一个虚拟机栈，其内部保存一个一个栈帧，对应每个方法。线程私有，生命周期与线程相同。

作用：主管Java程序的运行，保存方法的局部变量，部分结果，并参与方法的调用和返回。

栈中常见问题

① StackOverFlowError

如果采用固定大小的虚拟机栈，每个线程创建时会独立选定栈的大小，如果线程请求分配的栈容量超过虚拟机栈允许的最大容量就会报错。

StackOverflowError 是一个java中常出现的错误：在jvm运行时的数据区域中有一个java虚拟机栈，当执行java方法时会进行压栈弹栈的操作。在栈中会保存局部变量，操作数栈，方法出口等等。jvm规定了栈的最大深度，当执行时栈的深度大于了规定的深度，就会抛出StackOverflowError错误 （递归次数过多，栈帧就增多，整个操作数栈就会变大，可能报错）

②OutOfMemoryError（线程中空间不足 无法扩展虚拟机栈）

如果虚拟机栈可以动态扩展，并且申请扩展时无法申请到足够内存，或者创建的新的线程时没有足够的内存去创建对应的虚拟机栈，就会报错。

栈的执行原理

jvm直接对栈的操作有两个，就是对于栈帧的压栈和出栈

不同线程中所包含的栈帧是不允许相互引用的，即不可能在一个栈帧中引用另一个栈帧中的方法。

Java有两种返回函数的方式，一种是return指令，另一种是抛出异常。

局部变量表

也叫局部变量数组或本地变量表

定义为一个数字数组，主要用于存储方法参数和定义在方法内部的局部变量，

这些数据包括八种基本类型，对象引用，返回值类型。

由于局部变量表是建立在线程的栈上，是线程的私有数据，不存在数据安全问题，涉及不到共享问题。

局部变量表所需的容量是在编译器就确定下来,运行时不会改变。举例说明

关于Slot的理解

在局部变量表中，最基本的存储单位是Slot（变量槽）

32位以内的类型只占一个slot（包括返回值类型），64位的类型（long和double）占两个slot。

byte,short,char全部转化位int，boolean也转化为int ，0是false非0是true。引用类型占一个slot。

局部变量表0索引出还有一个this，因为这个方法不是静态方法，每个类都会含有构造器调用this关键字，除了静态方法不可调用this，其他非静态方法都可以调用，作用于全部非静态方法，所以每个0索引处都会有一个this参数。

slot重复利用

栈帧中的局部变量表中的槽位是可以重复利用的，如果一个局部变量过了其作用域，那么在其作用域之后声明的局部变量就很有可能会复用过期的局部变量的槽位，从而达到节省资源的目的。

局部变量表中的变量也是最重要的垃圾回收根节点，只要被局部变量表中直接或间接引用的对象都不会被回收。

操作数栈

每一个独立的栈帧中除了包含局部变量表之外，还包含一个后进先出的操作数栈，也可以称之为表达式栈。

在方法执行过程中，根据字节码指令，往栈中写入数据或提取数据，即入栈出栈。

当一个方法开始执行时，一个新的栈帧会随之被创建出来，这个方法的操作数栈式空的，数组（操作数栈）一旦创建，就会确定长度，其所需的最大长度在编译器就定义好了，保存code属性中，为max_stack值。

栈中任何一个元素都是可以任意的Java类型。

🌂32bit的类型占用一个栈单位深度

🌂64bit的类型占用两个栈单位深度

动态链接

作用：指向运行时常量池的引用

每个栈帧内部都包含一个指向运行时常量池中该栈帧所属方法的引用，包含这个引用的目的就是为了支持当前方法的代码能够实现动态链接。

当class文件加载到内存后，运行时常量池随之加载到方法区中，栈帧中的动态链接，存储运行时常量池的符号引用。根据符号引用去运行时常量池中寻找当前栈帧所属的方法。

动态语言和静态语言

动态类型语言和静态类型语言的区别是对于类型检查是在编译期还是在运行期，再概括一点就是，静态类型语言是判断变量自身的类型信息，动态类型语言始判断变量值的类型信息，变量没有类型信息，变量值才有。

例如：String name = “ygw”; 不可以写成 String name = ygw； 在编译器进行了类型检查，所以Java是静态语言

再比如：js语言，var name = “ygw”为字符类新 var name = 123 为整形

​ 在面向对象编程中，会频繁的使用到动态分配，如果在每次动态分派的过程中都要重新在类的方法元数据中搜索合适的目标的话就可能影响到执行效率。

如果说类的结构很深，当我们调用一个虚方法时，可能是最里面的父类所持有的方法，如果对此方法频繁调用，每次都会浪费一定的时间去在每个类中寻找这个方法，为了提高性能，JVM在类的方法区建立一个虚方法表。使用s索引表来代替查找。（每个类都有一个虚方法表）

方法返回地址

一个方法的执行后会有两种返回形式，一是执行引擎遇到任意方法返回的字节码指令，根据指令返回结果给上层调用者，称为正常返回。另一种是方法执行过程中遇到异常，并且这个异常没有再方法体中做处理（无论是虚拟机内部异常，还是抛出的异常），只要在方法表中没有搜到匹配的异常处理器，就会导致方法的退出，不会给上层调用者任何返回信息。

方法区

1.8同1.7比，最大的差别就是：元数据区取代了永久代。元空间的本质和永久代类似，都是对JVM规范中方法区的实现。不过元空间与永久代之间最大的区别在于：元数据空间并不在虚拟机中，而是使用本地内存。

​ 当JVM使用类装载器装载某个类时，它首先要定位对应的class文件，然后读入这个class文件，最后，JVM提取该文件的内容信息，并将这些信息存储到方法区，最后返回一个class实例。

🌂 方法区是什么？有哪些特点？

方法区是系统分配的一个内存逻辑区域，是用来存储类型信息的(类型信息可理解为类的描述信息)。方法区主要有以下几个特点：
**一.**方法区是线程安全的。由于所有的线程都共享方法区，所以，方法区里的数据访问必须被设计成线程安全的。例如，假如同时有两个线程都企图访问方法区中的同一个类，而这个类还没有被装入JVM，那么只允许一个线程去装载它，而其它线程必须等待
**二.**方法区的大小不必是固定的，JVM可根据应用需要动态调整。同时，方法区也不一定是连续的，方法区可以在一个堆(甚至是JVM自己的堆)中自由分配。
**三.**方法区也可被垃圾收集，当某个类不在被使用(不可触及)时，JVM将卸载这个类，进行垃圾收集

🌂 方法区里存放的是哪些内容？

方法区里存的都是类型信息，也就是类的信息，而类的信息又包括以下内容：
类的全限定名(类的全路径名)
类的直接超类的全限定名(如果这个类是Object,则它没有超类)
这个类是类型(类)还是接口
类的访问修饰符,如public、abstract、final等
所有的直接接口全限定名的有序列表(假如它实现了多个接口)
常量池
字段、方法信息、类变量信息(静态变量) 装载该类的装载器的引用(classLoader)、类型引用(class)

堆及垃圾回收下一篇~~~

本地方法栈

对于一个运行中的Java程序而言，它还可能会用到一些跟本地方法相关的数据区。当某个线程调用一个本地方法时，它就进入了一个全新的并且不再受虚拟机限制的世界。本地方法可以通过本地方法接口来访问虚拟机的运行时数据区，但不止如此，它还可以做任何它想做的事情。

　　本地方法本质上时依赖于实现的，虚拟机实现的设计者们可以自由地决定使用怎样的机制来让Java程序调用本地方法。

　　任何本地方法接口都会使用某种本地方法栈。当线程调用Java方法时，虚拟机会创建一个新的栈帧并压入Java栈。然而当它调用的是本地方法时，虚拟机会保持Java栈不变，不再在线程的Java栈中压入新的帧，虚拟机只是简单地动态连接并直接调用指定的本地方法。

什么是运行时数据区

Java虚拟机在执行Java程序得过程中会把它所管理得内存划分为若干个不同的数据区域，大致分为方法区、虚拟机栈、本地方法栈、堆、程序计数器。这些区域都有各自的用途，都有自己创建和销毁的时间。有的区域随着虚拟机的进程的启动而存在，比如方法区、堆。有的区域则是依赖用户线程的启动和结束而建立和销毁，比如虚拟机栈，程序计数器。

什么时是程序计数器

他是一块很小的内存空间，他的作用可以看作是当前线程所执行的字节码的行号指示器。在虚拟机的概念模型里，字节码解释器就是通过改变这个计数器的值来选取下一条需要执行的字节码指令，分支、循环、跳转、异常处理、线程恢复登基础功能都需要依赖这个计数器来完成。由于Java虚拟机的多线程是通过线程轮流切换并分配处理器执行时间的方式来实现的，任何一个时刻，一个处理器质会执行一条线程中的指令。因此为了线程切换回来后能够恢复到正确的执行位置，每个线程都会有单独的程序计数器，换个线程之间的计数器互不影响，独立存储，称这样的区域为线程私有区域。

什么是Java虚拟机栈

他与程序计数器一样，也是线程私有的，生命周期与线程相同，虚拟机栈描述的是Java方法执行的内存模型，每个方法执行的时候都会创建一个栈帧，用于存储局部变量表、操作数栈、动态链接、方法出口等信息，每个方法被调用完成就对应一个栈帧从虚拟机栈出栈。

什么是堆

堆是Java虚拟机中所管理的内存中最大的一块，他是一块被所有线程共享的一块区域，在虚拟机启动时创建，此内存区域的唯一目的就是存放对象实例。在Java虚拟机规范中描述的是：所有对象及数组都要在堆上分配。

堆也是垃圾收集器管理的主要区域，现在的垃圾回收器采用的分代收集算法，所以Java堆中还可以细分为新生代和老年代。在细致一点有Eden空间，from 幸存者区，to 幸存者区。无论任何划分，斗鱼存放的内容无关，无论哪个区域，存储的都是对象实例，进一步划分是为了更好挥手内存，或者更快的分配内存。

堆也是可以动态扩展的，Java虚拟机规范的规定，Java堆可以储在物理上不连续的内存空间，只要逻辑上是连续的即可，就像我们的磁盘空间一样，在实现时，既可以实现成固定大小，也可进行动态扩展（通过Xmx和Xms实现），如果当堆中没有内存分配实例，并且堆无法进行扩展时，就会抛出OOM异常。

什么是方法区

方法区和堆一样，都是线程共享的区域，它用于存储已被虚拟机加载的类信息、常量、静态变量等数据。Java虚拟机规范吧方法区描述为堆的一个逻辑部分。在HotSpot虚拟机上开发和部署程序的开发者来说，很多人成方法区为“永久代”。对于其他虚拟机来说不存在永久代这个概念。

Java虚拟机规范对这个区域限制非常宽松，他和堆一样不需要连续的内存空间和可扩展外，后还可以选择不实现垃圾收集，这个区域较少的出现垃圾回收，这个区域回收的目标是针对常量池的回收和对数据类型的卸载，可抛出OOM异常。

方法区垃圾回收

永久代的垃圾收集主要回收两部分内容:废弃常量和无用的类。回收废弃常量与回收Java堆中的对象非常类似。以常量池中字面量的回收为例,假如一个字符串“abc” 已经进入了常量池中,但是当前系统没有任何一个String对象是叫做“abc”的，换句话说 ,就是没有任何String对象引用常量池中的“abc”常量 ,也没有其他地方引用了这个字面量,如果这时发生内存回收,而且必要的话,这个“abc”常量就会被系统清理出常量池。常量池中的其他类(接口)、方法、字段的符号引用也与此类似。

判定一个常量是否是“废弃常量”比较简单,而要判定一个类是否是“无用的类”的条件则相对苛刻许多。类需要同时满足下面3个条件才能算是“无用的类”:

🌂 该类所有的实例都已经被回收,也就是Java堆中不存在该类的任何实例。
🌂 加载该类的ClassLoader已经被回收。
🌂 该类对应的java.lang.Class对象没有在任何地方被引用,无法在任何地方通过反射访问该类的方法。

运行时常量池

运行时常量池是方法区的一部分，Class文件中除了有类的版本、字段、方法、接口等信息外，还有以想信息是常量池，用于存放编译期生成的各种字面量和符号引用，这部分内容在类加载后存放到方法区的运行时常量池中。

对于运行时常量池，Java虚拟机规范没有做任何的细节要求，不同的提供商实现的虚拟机可以按照自己的需要来实现这个区域，运行时常量池会把直接引用也存储到运行时常量池，运行时常量池相对于Class文件常量池的另外一个重要特征时具备动态性，Java并不要求常量一定只在编译器产生，运行期间也可将新的常量放入池中，这种特性被开发人员利用得比较多得便是String类的intern()方法。当常量池无法申请到内存空间时会抛出OOM异常。

对象访问是如何进行的？

Object obj = new Objeect();

假设这个代码出现在方法体中，Object obj 这部分你语句将反映到Java栈的本地变量表中，作为一个reference类型数据出现。而new Object() 则反映到Java堆中，形成一块存储了Object类型的所有实例数据值的结构化内存，根据具体类型以及虚拟机实现的兑现该内存布局的不同，这块内存的长度是不固定的。并且Java堆中还要包含能查找此对象类型的数据的地址信息，也就是类信息（包括：对象类型、父类、实现的接口、方法等）。存储在方区中。
